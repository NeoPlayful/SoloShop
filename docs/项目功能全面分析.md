# 📊 SoloShop 项目功能全面分析

## 一、核心功能模块

### 1. 前台商城（Storefront）

面向买家的商品展示与下单入口，重点是简单、快速、低门槛购买。

- 首页商品展示
- 商品分类展示
- 商品搜索与筛选
- 商品详情页
- 商品封面图展示
- 商品价格展示
- 商品库存状态展示
- 商品销量展示
- 购买须知展示
- 移动端响应式适配
- 站点公告展示
- 联系方式展示

### 2. 商品管理（Products）

用于后台维护数字商品、卡密商品、固定文本商品和人工发货商品。

- 商品新增
- 商品编辑
- 商品删除 / 软删除
- 商品上架 / 下架
- 商品隐藏销售
- 商品分类绑定
- 商品封面上传
- 商品详情编辑
- 商品价格设置
- 商品原价设置
- 商品成本价设置
- 最小购买数量设置
- 最大购买数量设置
- 商品排序设置
- 商品复制功能
- 商品销量统计
- 商品库存统计

### 3. 商品分类（Categories）

用于对商品进行归类，方便前台展示和后台管理。

- 分类新增
- 分类编辑
- 分类删除
- 分类排序
- 分类显示 / 隐藏
- 分类商品数量统计
- 分类 Slug 管理

### 4. 卡密库存（Cards / Inventory）

SoloShop 的核心模块之一，用于管理自动发货的卡密库存。

- 批量导入卡密
- 单条新增卡密
- 卡密重复检测
- 卡密批次管理
- 卡密状态管理
- 卡密库存统计
- 卡密搜索与筛选
- 卡密禁用 / 启用
- 卡密批量删除
- 售出卡密绑定订单
- 卡密完整内容查看
- 卡密查看操作日志
- 库存不足提醒
- 库存锁定机制
- 订单超时释放库存

**卡密状态：**

- available：可售
- locked：已锁定，等待支付
- sold：已售出
- disabled：已禁用
- expired：已过期

### 5. 订单系统（Orders）

负责订单创建、状态流转、支付关联、发货关联和售后处理。

- 游客下单
- 订单号生成
- 订单金额计算
- 商品快照记录
- 买家邮箱记录
- 买家联系方式记录
- 买家 IP 记录
- User-Agent 记录
- 订单状态管理
- 支付状态管理
- 发货状态管理
- 订单超时关闭
- 订单查询
- 订单详情查看
- 订单备注管理
- 手动关闭订单
- 手动标记支付成功
- 重新发货
- 手动发货
- 订单操作日志

**订单状态建议拆分：**

- orderStatus：pending / completed / cancelled / closed
- paymentStatus：unpaid / paid / failed / refunded
- deliveryStatus：pending / delivering / delivered / failed
- refundStatus：none / partial / full

### 6. 支付系统（Payments）

负责创建支付、接收支付回调、校验支付结果和触发自动发货。

- 支付渠道管理
- 支付渠道启用 / 禁用
- 支付参数配置
- 支付链接生成
- 支付二维码生成
- 支付状态查询
- 支付回调接收
- 支付签名验证
- 支付金额校验
- 支付平台订单号记录
- 支付回调日志
- 重复回调幂等处理
- 支付异常记录
- 后台手动补单

**MVP 推荐优先支持：**

- 易支付
- Stripe Checkout
- 手动补单

### 7. 自动发货（Delivery）

支付成功后自动交付商品内容，是发卡系统最关键的稳定性模块。

- 卡密自动发货
- 固定文本发货
- 人工发货
- 发货记录生成
- 发货状态更新
- 发货失败记录
- 发货失败重试
- 重新发送原发货内容
- 邮件发货通知
- 支付成功页展示发货内容
- 后台查看发货内容
- 发货内容脱敏显示

**重要规则：**

- 已成功发货的订单，重新发货时不能重新扣一张卡密
- 重新发货只能发送原 delivery 记录中的内容
- 支付回调重复时不能重复发货

### 8. 订单查询（Order Lookup）

用于买家在前台查询自己的订单和发货内容。

- 订单号查询
- 邮箱二次校验
- 支付状态展示
- 发货状态展示
- 发货内容展示
- 订单创建时间展示
- 支付时间展示
- 发货时间展示
- 重发发货邮件
- 查询频率限制
- 防止订单号泄露导致卡密泄露

## 四、安全与风控

### 1. 前台风控

降低刷单、恶意查询、库存占用和爬虫风险。

- 创建订单限流
- 查询订单限流
- 重发邮件限流
- IP 黑名单
- 邮箱黑名单
- User-Agent 记录
- 验证码支持
- 订单超时自动关闭
- 锁定库存自动释放

### 2. 后台安全

保护管理后台和敏感配置。

- HTTP-only Cookie
- SameSite Cookie
- Secure Cookie
- JWT 过期时间
- 登录失败限制
- 管理员密码加密
- 支付密钥脱敏
- SMTP 密码脱敏
- 卡密默认脱敏
- 查看完整卡密记录日志
- 敏感操作二次确认
- 管理员权限预留

### 3. 幂等与一致性

保证支付回调、库存扣减和自动发货不会重复执行。

- 支付回调幂等锁
- 订单发货幂等锁
- 平台订单号唯一约束
- 订单号唯一约束
- 卡密库存状态流转
- 数据库事务处理
- Redis 分布式锁
- 支付金额二次校验
- 支付签名验证
- 重复回调直接返回成功

**主要模块：** 前台商城、商品管理、分类管理、卡密库存、订单系统、支付系统、自动发货、订单查询、后台管理、系统设置、安全风控、操作日志。

**Phase 1 核心目标：**
- 商品能展示
- 买家能下单
- 支付能回调
- 库存不超卖
- 卡密能自动发货
- 订单能查询
- 后台能补单和重新发货

**SoloShop MVP 的关键判断标准：**

> 不是功能数量越多越好，而是交易链路必须稳定。第一版只要把商品、订单、支付、库存、发货、后台补救做好，就已经是一个合格的发卡系统 MVP。
