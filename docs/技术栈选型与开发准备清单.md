## 一、技术栈选型

| 层级 | 选型 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | React | 19 | 前台商城 + 后台管理界面 |
| 构建工具 | Vite | 8 | 前端开发与构建 |
| 语言 | TypeScript | ~5 | 前后端统一类型安全 |
| 样式 | Tailwind CSS | 4 | Utility-first，适合快速搭建管理后台 |
| 状态管理 | TanStack Query | 5 | 服务端数据缓存、请求状态管理 |
| 客户端状态 | Zustand | 5 | 登录状态、主题、侧边栏等轻量状态 |
| 路由（前端） | react-router-dom | 7 | 前台商城与后台路由 |
| 图标 | Heroicons / Lucide React | 2.2 / latest | 统一图标库 |

| 后端框架 | Fastify | 5 | Node.js 高性能 API 框架 |
| ORM | Prisma | 5.22+ | PostgreSQL ORM，支持迁移和类型生成 |

## 三、开发环境设置

```bash
# 1. 克隆项目
git clone <your-repo-url> SoloShop
cd SoloShop

# 2. 准备环境变量
cp .env.example .env

# 3. 启动 PostgreSQL 和 Redis
docker compose up -d postgres redis

# 4. 安装后端依赖
cd server
npm install

# 5. 初始化数据库
npx prisma generate
npx prisma migrate dev
npm run seed

# 6. 启动后端
npm run dev
# Fastify API: http://localhost:4000

# 7. 安装前端依赖
cd ../frontend
npm install

# 8. 启动前端
npm run dev
# React Vite: http://localhost:5173
```

**生产环境：**
```bash
# 构建并启动全部服务
docker compose up -d --build

# 查看日志
docker compose logs -f

# 停止服务
docker compose down
```

## 五、安全措施

- ✅ 管理员密码 bcrypt 加密存储
- ✅ JWT 使用 httpOnly cookie，避免前端 JS 直接读取 Token
- ✅ Cookie 设置 SameSite=Lax，生产环境启用 Secure
- ✅ 后台写操作预留 CSRF 防护
- ✅ 登录失败次数限制，防止后台爆破
- ✅ Redis 限流：创建订单、订单查询、重发邮件、后台登录
- ✅ 支付回调签名验证，禁止伪造支付成功
- ✅ 支付金额校验，防止小额支付冒充大额订单
- ✅ 支付回调幂等锁，防止重复回调重复发货
- ✅ 自动发货幂等，已发货订单只允许重发原内容
- ✅ 卡密库存状态机：available、locked、sold、disabled、expired
- ✅ 查看完整卡密必须记录操作日志
- ✅ 支付密钥、SMTP 密码后台脱敏显示
- ✅ 商品成本价、后台备注不返回前台接口
- ✅ 订单查询要求订单号 + 邮箱，防止卡密泄露
- ✅ 上传文件限制类型和大小
- ✅ 统一错误响应格式，避免暴露系统堆栈
- ✅ Nginx 限制上传大小和代理超时
- ✅ 数据库定期备份，避免误删和数据损坏

```yaml

services:
  postgres:   # PostgreSQL 16，核心数据库
```



## 十、总结

| 类型 | 数量 / 状态 |
|------|-------------|
| 前端核心页面 | 10+ 个 |
| 后台核心页面 | 12+ 个 |
| MVP API 接口 | 45+ 个 |
| 数据库核心表 | 15+ 张 |
| Docker 服务 | 5 个 |
| 必做核心链路 | 商品 → 下单 → 支付 → 回调 → 发货 → 查询 |
| 推荐上线方式 | 单商户 Docker Compose 部署 |
| 后期增强方向 | 用户体系、优惠码、代理返佣、插件化 |

**主要模块：** 前台商城、后台管理、商品管理、卡密库存、订单系统、支付系统、自动发货、系统设置、安全日志

**Phase 1 MVP 目标：**
- 商品能创建和上架
- 用户能下单和支付
- 支付回调能正确处理
- 库存不会超卖
- 卡密能自动发货
- 后台能补单、重发、关闭订单
- 操作过程可追踪、可审计

**关键质量标准：**
- 支付不丢单
- 回调不重复发货
- 库存不超卖
- 卡密不泄露
- 后台可补救
- 日志可追踪
