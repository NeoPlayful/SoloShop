# 项目目录结构

## 仓库概览

```
SoloShop/
├── server/                 # Fastify 后端 API (:4000)
├── frontend/               # React 19 + Vite 前端 (:5173)
├── nginx/                  # Nginx 反向代理配置
├── docker-compose.yml      # 4个服务编排 (nginx + api + postgres + redis)
└── docs/                   # 项目文档
```

---

## server — Fastify 后端

```
server/
├── src/
│   ├── app.ts              # 入口文件，注册插件和路由
│   ├── routes/             # API 路由
│   │   ├── auth.ts             # 认证：登录 / 退出 / 获取当前用户 / 修改密码
│   │   │                     # 统一 /api/auth/*，管理员和未来用户共用，以权限区分
│   │   ├── public/         # 前台公开接口（无需认证）
│   │   │   ├── categories.ts    # 商品分类列表
│   │   │   ├── products.ts      # 商品列表 & 详情
│   │   │   ├── orders.ts        # 创建订单 / 查询订单 / 订单状态
│   │   │   ├── payment.ts       # 支付渠道列表 / 创建支付
│   │   │   └── settings.ts      # 站点配置 / 公告 / 联系方式
│   │   ├── admin/          # 后台管理接口（需管理员权限）
│   │   │   ├── dashboard.ts    # 仪表板概览 / 销售趋势 / 排行
│   │   │   ├── categories.ts   # 分类 CRUD / 排序 / 启用禁用
│   │   │   ├── products.ts     # 商品 CRUD / 上下架 / 复制 / 统计
│   │   │   ├── cards.ts        # 卡密管理 / 批量导入 / 批次管理
│   │   │   ├── orders.ts       # 订单列表 / 详情 / 补单 / 重发 / 关闭
│   │   │   ├── payments.ts     # 支付记录 / 回调日志 / 手动补单
│   │   │   ├── deliveries.ts   # 发货记录 / 重试失败 / 手动发货
│   │   │   ├── payment-channels.ts  # 支付渠道配置 / 启用禁用
│   │   │   ├── settings.ts     # 系统设置（站点/订单/安全/邮件）
│   │   │   ├── admins.ts       # 管理员管理 / 角色权限
│   │   │   ├── logs.ts         # 操作日志 / 登录日志 / 安全日志
│   │   │   └── security.ts     # 限流配置 / IP黑名单 / 邮箱黑名单
│   │   ├── webhook/         # 外部支付回调
│   │   │   └── pay.ts           # 支付 Webhook（易支付/Stripe）
│   │   └── system/
│   │       └── health.ts        # 健康检查
│   ├── plugins/             # 业务插件系统（预留，未来支持插件化扩展）
│   └── lib/                 # 共享工具库
│       ├── auth.ts             # JWT 生成/验证中间件
│       ├── db.ts               # Prisma 客户端单例
│       ├── api-utils.ts        # 统一错误处理 / 响应格式
│       ├── card-pool.ts        # 卡密池：锁定 / 扣减 / 释放 / 统计
│       ├── delivery.ts         # 自动发货引擎：发货 / 重试 / 邮件通知
│       ├── payment/            # 支付模块
│       │   ├── index.ts        # 支付统一入口
│       │   ├── yipay.ts        # 易支付对接
│       │   └── stripe.ts       # Stripe 对接
│       ├── email.ts            # 邮件发送（SMTP）
│       ├── cache.ts            # Redis 缓存封装
│       ├── rate-limit.ts       # 限流工具
│       └── upload.ts           # 文件上传（商品封面/Logo）
├── prisma/
│   ├── schema.prisma           # 数据模型定义
│   └── seed.ts                 # 种子数据脚本
├── uploads/                    # 上传文件本地存储
├── .env                        # 环境变量
├── Dockerfile
├── package.json
└── tsconfig.json
```

```
nginx/
```

## 服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx | 3000 | 统一入口，反向代理 |
| Fastify 后端 | 4000 | API 服务 |
| React 前端 | 5173 | 开发服务器（Vite） |
| PostgreSQL | 5432 | 数据库 |
| Redis | 6379 | 缓存 / 限流 / 分布式锁 |

## 核心业务模块与对应目录

| 业务模块 | 后端 routes | 后端 lib | 前端 pages |
|---------|------------|---------|-----------|
| 前台商店 | `public/products`, `public/categories` | — | `StorePage`, `ProductPage` |
| 购物下单 | `public/orders` | `card-pool.ts` | `CheckoutPage` |
| 支付 | `public/payment`, `pay/callback` | `payment/` | `PayPage` |
| 自动发货 | — | `delivery.ts`, `email.ts` | `OrderSuccessPage` |
| 订单查询 | `public/orders` | — | `OrderQueryPage`, `OrderDetailPage` |
| 商品管理 | `admin/products`, `admin/categories` | `upload.ts` | `ProductsPage`, `CategoriesPage` |
| 卡密库存 | `admin/cards` | `card-pool.ts` | `CardsPage`, `CardsImportPage` |
| 订单管理 | `admin/orders` | — | `OrdersPage` |
| 支付管理 | `admin/payments`, `admin/payment-channels` | `payment/` | `PaymentsPage`, `PaymentChannelsPage` |
| 发货管理 | `admin/deliveries` | `delivery.ts` | `DeliveriesPage` |
| 系统设置 | `admin/settings` | — | `SettingsPage` |
